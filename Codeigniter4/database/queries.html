쿼리 기본 사항

참고

CodeIgniter는 테이블 및 열 이름에 점(.)을 지원하지 않습니다. v4.5.0부터 점이 포함된 데이터베이스 이름을 지원합니다.

일반 쿼리

$db->query()

쿼리를 제출하려면 query() 메서드를 사용하세요:

<?php

$db = db_connect();
$db->query('YOUR QUERY HERE');

query() 메서드는 “읽기” 유형 쿼리가 실행될 때 데이터베이스 결과 객체를 반환합니다. 이 결과는 결과 표시에 사용할 수 있습니다. “쓰기” 유형의 쿼리가 실행되면 성공 또는 실패에 따라 단순히 true 또는 false를 반환합니다. 데이터를 검색할 때 일반적으로 다음과 같이 쿼리를 자신의 변수에 할당합니다.

<?php

$query = $db->query('YOUR QUERY HERE');

참고

OCI8 드라이버를 사용하는 경우 SQL 문은 세미콜론(;)으로 끝나서는 안 됩니다. PL/SQL 문은 세미콜론(;)으로 끝나야 합니다.

단순화된 쿼리

$db->simpleQuery()

simpleQuery() 메서드는 $db->query() 메서드를 단순화한 버전입니다. 데이터베이스 결과 세트를 반환하지 않으며, 쿼리 타이머를 설정하거나, 바인드 데이터를 컴파일하거나, 디버깅을 위해 쿼리를 저장하지 않습니다. 간단히 쿼리를 제출할 수 있습니다. 대부분의 사용자는 이 기능을 거의 사용하지 않습니다.

이는 데이터베이스 드라이버의 “실행” 기능이 반환하는 모든 것을 반환합니다. 이는 일반적으로 INSERT, DELETE 또는 UPDATE 문(실제로 사용해야 하는 용도)과 같은 쓰기 유형 쿼리의 성공 또는 실패 시 true/false이고, 가져오기 가능한 결과가 있는 쿼리의 성공 시 리소스/객체입니다.

<?php

if ($db->simpleQuery('YOUR QUERY')) {
    echo 'Success!';
} else {
    echo 'Query failed!';
}

참고

PostgreSQL의 pg_exec() 함수(예를 들어)는 쓰기 유형 쿼리의 경우에도 성공 시 항상 리소스를 반환합니다. 따라서 부울 값을 찾고 있다면 이 점을 염두에 두십시오.

수동으로 데이터베이스 접두사 작업

$db->prefixTable()

데이터베이스 접두사를 설정했고, 예를 들어 기본 SQL 쿼리에서 사용할 테이블 이름 앞에 이를 붙이고 싶다면 다음을 사용할 수 있습니다.

<?php

$db->prefixTable('tablename'); // outputs prefix_tablename

$db->setPrefix()

어떤 이유로든 새 연결을 만들지 않고 프로그램적으로 접두사를 변경하고 싶다면 이 메서드를 사용할 수 있습니다.

<?php

$db->setPrefix('newprefix_');
$db->prefixTable('tablename'); // outputs newprefix_tablename

$db->getPrefix()

이 메서드로 언제든지 현재 접두사를 가져올 수 있습니다:

<?php

$DBPrefix = $db->getPrefix();

식별자 보호

$db->protectIdentifiers()

많은 데이터베이스에서는 테이블 및 필드 이름을 보호하는 것이 좋습니다(예: MySQL의 백틱 사용). 쿼리 빌더 쿼리는 자동으로 보호됩니다. 그러나 식별자를 수동으로 보호해야 하는 경우 다음을 사용할 수 있습니다.

<?php

$db->protectIdentifiers('table_name');

중요

쿼리 빌더는 사용자가 제공하는 필드 및 테이블 이름을 적절하게 인용하기 위해 최선을 다합니다. 임의의 사용자 입력과 함께 작동하도록 설계되지 않았습니다. 정리되지 않은 사용자 데이터를 제공하지 마십시오.

이 함수는 데이터베이스 설정 파일에 접두사가 지정되어 있다고 가정하여 테이블에 접두사를 추가합니다. 접두사 적용을 활성화하려면 두 번째 매개변수에 true(부울)를 설정하세요:

<?php

$db->protectIdentifiers('table_name', true);

이스케이프 값

데이터를 데이터베이스에 제출하기 전에 데이터를 이스케이프 처리하는 것은 매우 좋은 보안 관행입니다. CodeIgniter에는 이를 수행하는 데 도움이 되는 세 가지 방법이 있습니다.

1. $db->escape()

이 함수는 문자열 데이터만 이스케이프할 수 있도록 데이터 유형을 결정합니다. 또한 데이터 주위에 작은따옴표를 자동으로 추가하므로 다음을 수행할 필요가 없습니다.

<?php

$sql = 'INSERT INTO table (title) VALUES(' . $db->escape($title) . ')';

2. $db->escapeString()

이 함수는 유형에 관계없이 전달된 데이터를 이스케이프합니다. 대부분의 경우 이 기능보다는 위의 기능을 사용하게 됩니다. 다음과 같이 기능을 사용하십시오.

<?php

$sql = "INSERT INTO table (title) VALUES('" . $db->escapeString($title) . "')";

3. $db->escapeLikeString()

이 방법은 문자열의 LIKE 와일드카드(%, _)도 적절하게 이스케이프되도록 LIKE 조건에서 문자열을 사용할 때 사용해야 합니다.

<?php

$search = '20% raise';
$sql    = "SELECT id FROM table WHERE column LIKE '%" . $db->escapeLikeString($search) . "%' ESCAPE '!'";

중요

escapeLikeString() 메소드는 느낌표를 사용하여 LIKE 조건의 특수 문자를 이스케이프합니다. 이 방법은 사용자가 직접 따옴표로 묶은 부분 문자열을 이스케이프하기 때문에 ESCAPE 조건을 자동으로 추가할 수 없으므로 수동으로 추가해야 합니다.

쿼리 바인딩

바인딩을 사용하면 시스템에서 쿼리를 함께 구성하여 쿼리 구문을 단순화할 수 있습니다. 다음 예를 고려하십시오.

<?php

$sql = 'SELECT * FROM some_table WHERE id = ? AND status = ? AND author = ?';
$db->query($sql, [3, 'live', 'Rick']);

쿼리의 물음표는 쿼리 함수의 두 번째 매개변수에 있는 배열의 값으로 자동으로 대체됩니다.

바인딩은 IN 세트로 변환되는 배열에서도 작동합니다.

<?php

$sql = 'SELECT * FROM some_table WHERE id IN ? AND status = ? AND author = ?';
$db->query($sql, [[3, 6], 'live', 'Rick']);

결과 쿼리는 다음과 같습니다:

SELECT * FROM some_table WHERE id IN (3,6) AND status = 'live' AND author = 'Rick'

바인드 사용의 두 번째 이점은 값이 자동으로 이스케이프되어 더 안전한 쿼리를 생성한다는 것입니다. 데이터를 수동으로 이스케이프하는 것을 기억할 필요가 없습니다. 엔진이 자동으로 데이터를 이스케이프 처리합니다.

명명된 바인딩

바인딩된 값의 위치를 ​​표시하기 위해 물음표를 사용하는 대신 바인딩 이름을 지정하여 전달된 값의 키가 쿼리의 자리 표시자와 일치하도록 할 수 있습니다.

<?php

$sql = 'SELECT * FROM some_table WHERE id = :id: AND status = :status: AND author = :name:';
$db->query($sql, [
    'id'     => 3,
    'status' => 'live',
    'name'   => 'Rick',
]);

참고

쿼리의 각 이름은 반드시 콜론으로 묶어야 합니다.

오류 처리

$db->error()

발생한 마지막 오류를 가져와야 한다면 error() 메서드는 해당 코드와 메시지가 들어 있는 배열을 반환합니다. 간단한 예는 다음과 같습니다:

<?php

if (! $db->simpleQuery('SELECT `example_field` FROM `example_table`')) {
    $error = $db->error(); // Has keys 'code' and 'message'
}

준비된 쿼리

대부분의 데이터베이스 엔진은 쿼리를 한 번 준비한 다음 새 데이터 집합을 사용하여 해당 쿼리를 여러 번 실행할 수 있는 일종의 준비된 문 형식을 지원합니다. 이렇게 하면 데이터가 쿼리 자체와 다른 형식으로 데이터베이스에 전달되므로 SQL 주입 가능성이 제거됩니다. 동일한 쿼리를 여러 번 실행해야 하는 경우에도 훨씬 더 빠를 수 있습니다. 그러나 모든 쿼리에 이를 사용하면 데이터베이스를 두 배 더 자주 호출하게 되므로 성능에 큰 타격을 줄 수 있습니다. 쿼리 작성기 및 데이터베이스 연결이 이미 데이터 이스케이프를 처리하므로 안전 측면도 이미 처리되어 있습니다. 그러나 준비된 문이나 준비된 쿼리를 실행하여 쿼리를 최적화하는 기능이 필요한 경우도 있습니다.

쿼리 준비

이는 prepare() 메소드를 사용하여 쉽게 수행할 수 있습니다. 이는 쿼리 객체를 반환하는 클로저인 단일 매개변수를 사용합니다. 쿼리 개체는 삽입, 업데이트, 삭제, 교체 및 가져오기를 포함한 모든 “최종” 유형 쿼리에 의해 자동으로 생성됩니다. 이는 쿼리 빌더를 사용하여 쿼리를 실행하면 가장 쉽게 처리됩니다. 쿼리는 실제로 실행되지 않으며 값은 적용되지 않고 자리 표시자 역할을 하므로 중요하지 않습니다. 그러면 ReadyQuery 객체가 반환됩니다.

<?php

$pQuery = $db->prepare(static fn ($db) => $db->table('user')->insert([
    'name'    => 'x',
    'email'   => 'y',
    'country' => 'US',
]));

쿼리 빌더를 사용하지 않으려면 값 자리 표시자에 물음표를 사용하여 수동으로 쿼리 개체를 생성할 수 있습니다.

<?php

use CodeIgniter\Database\Query;

$pQuery = $db->prepare(static function ($db) {
    $sql = 'INSERT INTO user (name, email, country) VALUES (?, ?, ?)';

    return (new Query($db))->setQuery($sql);
});

데이터베이스가 문 준비 단계 동안 전달된 옵션 배열을 요구하는 경우 두 번째 매개변수를 통해 해당 배열을 전달할 수 있습니다.

<?php

use CodeIgniter\Database\Query;

$pQuery = $db->prepare(static function ($db) {
    $sql = 'INSERT INTO user (name, email, country) VALUES (?, ?, ?)';

    return (new Query($db))->setQuery($sql);
}, $options);

참고

현재 실제로 옵션 배열을 사용하는 유일한 데이터베이스는 SQLSRV입니다.

쿼리 실행

준비된 쿼리가 있으면 execute() 메서드를 사용하여 실제로 쿼리를 실행할 수 있습니다. 쿼리 매개변수에 필요한 만큼의 변수를 전달할 수 있습니다. 전달하는 매개변수 수는 쿼리의 자리 표시자 수와 일치해야 합니다. 또한 원래 쿼리에 나타나는 자리 표시자와 동일한 순서로 전달되어야 합니다.

<?php

// Prepare the Query
$pQuery = $db->prepare(static fn ($db) => $db->table('user')->insert([
    'name'    => 'x',
    'email'   => 'y',
    'country' => 'US',
]));

// Collect the Data
$name    = 'John Doe';
$email   = 'j.doe@example.com';
$country = 'US';

// Run the Query
$results = $pQuery->execute($name, $email, $country);

“쓰기” 유형의 쿼리의 경우 true 또는 false를 반환하여 쿼리의 성공 또는 실패를 나타냅니다. “읽기” 유형의 쿼리의 경우 표준 결과 세트를 반환합니다.

다른 방법

이러한 두 가지 기본 메서드 외에도 준비된 쿼리 개체에는 다음과 같은 메서드도 있습니다.

close()

PHP는 데이터베이스와 함께 열려 있는 모든 문을 닫는 작업을 꽤 잘 수행하지만 작업이 끝나면 항상 준비된 문을 닫는 것이 좋습니다.

<?php

if ($pQuery->close()) {
    echo 'Success!';
} else {
    echo 'Deallocation of prepared statements failed!';
}

참고

v4.3.0부터 close() 메소드는 모든 DBMS에서 준비된 명령문의 할당을 해제합니다. 이전에는 Postgre, SQLSRV 및 OCI8에서 할당이 취소되지 않았습니다.

getQueryString()

그러면 준비된 쿼리가 문자열로 반환됩니다.

hasError()

마지막 execute() 호출에서 오류가 발생한 경우 부울 true/false를 반환합니다.

getErrorCode()

getErrorMessage()

오류가 발생한 경우 이러한 방법을 사용하여 오류 코드와 문자열을 검색할 수 있습니다.

쿼리 개체 작업

내부적으로 모든 쿼리는 CodeIgniter\Database\Query의 인스턴스로 처리되고 저장됩니다. 이 클래스는 매개변수 바인딩, 쿼리 준비, 쿼리에 대한 성능 데이터 저장을 담당합니다.

$db->getLastQuery()

마지막 Query 객체만 검색해야 하는 경우 getLastQuery() 메서드를 사용하세요.

<?php

$query = $db->getLastQuery();
echo (string) $query;

쿼리 클래스

각 쿼리 개체는 쿼리 자체에 대한 여러 정보를 저장합니다. 이는 타임라인 기능에서 부분적으로 사용되지만 귀하도 사용할 수 있습니다.

getQuery()

모든 처리가 완료된 후 최종 쿼리를 반환합니다. 다음은 데이터베이스로 전송된 정확한 쿼리입니다.

<?php

$sql = $query->getQuery();

Query 개체를 문자열로 캐스팅하여 동일한 값을 검색할 수 있습니다.

<?php

$sql = (string) $query;

getOriginalQuery()

객체에 전달된 원시 SQL을 반환합니다. 여기에는 바인딩이나 접두사가 교체되지 않습니다.

<?php

$sql = $query->getOriginalQuery();

hasError()

이 쿼리를 실행하는 동안 오류가 발생하면 이 메서드는 true를 반환합니다.

<?php

if ($query->hasError()) {
    echo 'Code: ' . $query->getErrorCode();
    echo 'Error: ' . $query->getErrorMessage();
}

isWriteType()

쿼리가 쓰기 유형 쿼리(예: INSERT, UPDATE, DELETE 등)인 것으로 확인된 경우 true를 반환합니다.

<?php

if ($query->isWriteType()) {
    // ... do something
}

swapPrefix()

하나의 테이블 접두사를 SQL의 다른 값으로 바꿉니다. 첫 번째 매개변수는 바꾸려는 원래 접두사이고, 두 번째 매개변수는 바꾸려는 값입니다.

<?php

$sql = $query->swapPrefix('ci3_', 'ci4_');

getStartTime()

쿼리가 실행된 시간을 마이크로초 단위로 가져옵니다.

<?php

$microtime = $query->getStartTime();

getDuration()

마이크로초 단위로 쿼리 기간을 초 단위로 나타내는 부동 소수점을 반환합니다.

<?php

$microtime = $query->getDuration();
